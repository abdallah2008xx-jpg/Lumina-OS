# AhmadOS Brand System

## Brand Thesis
AhmadOS should feel calm, premium, and usable within minutes.
The visual language should suggest trust first, personality second.

## Brand Pillars
- calm surfaces instead of busy decoration
- premium spacing instead of heavy ornament
- Arabic-first friendliness without isolating English users
- familiar navigation patterns with refined presentation
- one clear accent family, not many competing colors

## Shape Language
- outer application radius: `28px`
- card radius: `22px`
- input radius: `16px`
- pill radius: `999px`
- shadows should be soft and spread wide, never harsh black

## Layout Rhythm
- base spacing unit: `8px`
- major section spacing: `32px`
- hero spacing: `48px`
- comfortable content width: `1200px`
- default UI density should prefer clarity over compactness

## Motion
- hover transition: `160ms`
- panel transition: `240ms`
- page reveal: `420ms`
- use opacity, scale, and slight vertical drift
- avoid springy or playful motion in core system surfaces

## Wallpaper Direction
- use environmental gradients, mineral textures, or soft horizon forms
- avoid neon, cyberpunk, or over-saturated abstract art
- release wallpapers should support both login contrast and desktop calm
- keep one hero wallpaper family per release cycle

## Icon and Illustration Direction
- rounded geometric silhouettes
- muted fills with selective copper or blue highlights
- avoid mixing icon packs with different stroke languages
- onboarding illustrations should look editorial, not childish

## Bilingual Rules
- Arabic must look native, not retrofitted
- support RTL layouts for first-run, settings summaries, and update notes
- English should remain available as a full peer language
- mixed Arabic and Latin content should use generous spacing

## Surface Hierarchy
1. background atmosphere
2. elevated application shell
3. section cards
4. interactive rows and controls
5. accent pills and primary actions

## Default Component Tone
- primary button: Ahmad Blue fill with soft highlight
- secondary button: translucent slate with visible border
- destructive button: muted red with explicit confirmation copy
- toggles: rounded, high-contrast, and easy to scan
- badges: small, calm, and meaning-first

## Release Identity Guidance
- `dev`: cool blue
- `alpha`: copper
- `beta`: mist + blue
- `stable`: Ahmad Blue with ivory surfaces

## Immediate Output Targets
- branded wallpaper family
- welcome app visual shell
- update center visual shell
- settings landing shell
- lightly branded SDDM theme
