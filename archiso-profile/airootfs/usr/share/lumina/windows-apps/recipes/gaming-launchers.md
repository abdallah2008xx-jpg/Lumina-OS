# Gaming Launchers Launch Recipe

## Daily Start

- Start from the Linux-native launcher or Proton route first.
- Use the Windows workspace only for launchers, patchers, or utilities that still resist the Linux path.
- Keep the launcher support path separate from the claims about supported titles.

## Daily Rhythm

- Treat the Windows side as a fallback support layer, not as the default gaming promise.
- Keep launcher installers, caches, and notes organized so the workflow stays recoverable.
- Validate the launcher itself before claiming anything about the game behind it.

## Daily Close

- Close launchers cleanly and keep notes about what worked and what stayed unsupported.
- Move important logs or exported settings back into the Lumina workspace.
