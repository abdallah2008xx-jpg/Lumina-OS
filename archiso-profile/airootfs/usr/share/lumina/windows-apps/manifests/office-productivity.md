# Office and Productivity Workflow Manifest

## Linux-First Baseline

- Prefer Linux-native apps, browser workflows, and web-first Microsoft 365 usage where it already feels smooth.
- Use Lumina as the visible shell for documents, notes, meetings, and chat.
- Keep the Windows path as an escalation step, not the first habit.

## Windows Workspace Baseline

- Use a small, clean Windows workspace only when a document or tool still needs it.
- Keep the Windows VM lightweight and focused on the exact productivity gaps that remain.
- Avoid turning a simple Office workflow into a full creator-class VM unless the workload really grows.

## First Session Checklist

- Validate browser and Linux-native options first.
- Confirm whether Office web apps already solve the real need.
- If a Windows VM is still needed, keep it simple and document-focused.
- Test printing, document exchange, and cloud sync before expanding the toolset.

## Risks

- It is easy to overbuild this workflow. Lumina should stay Linux-first here unless a real blocker remains.
