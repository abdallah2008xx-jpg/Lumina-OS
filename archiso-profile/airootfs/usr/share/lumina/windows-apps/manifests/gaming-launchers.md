# Gaming Launchers Workflow Manifest

## Linux-First Baseline

- Prefer Heroic, Steam, Proton, Lutris, and Linux-native launch paths first.
- Keep Lumina as the main visible gaming shell.
- Use the Windows path only for launchers or workflows that still resist the Linux route.

## Windows Workspace Baseline

- Use a fallback Windows workspace for launcher reliability, patching, or utilities that do not behave well elsewhere.
- Keep the Windows workspace focused on launchers and support tools, not on pretending every title is officially supported there.
- Treat the Windows VM as a support layer, not the first marketing promise.

## First Session Checklist

- Confirm whether the launcher already has a good Linux-native route.
- If not, install only the launcher and one target workflow first.
- Keep game storage and cache placement intentional.
- Separate “launcher works” from “game is officially supported” in testing notes.

## Risks

- Launcher success does not guarantee full title success.
- Lumina does not promise anti-cheat bypass or stealth behavior.
