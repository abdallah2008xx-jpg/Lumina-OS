# Studio and Audio Workflow Manifest

## Linux-First Baseline

- Keep project storage, references, and general desktop work on Lumina.
- Treat Wine as a limited experiment path for DAWs and plugins, not the default promise.
- Validate MIDI, audio devices, and file flow before expanding the workflow.

## Windows Workspace Baseline

- Use one stable Windows VM for the audio workflow instead of splitting tools across many environments.
- Keep the VM lean and centered on the DAW and plugins that actually matter.
- Favor predictable storage and device mapping over flashy complexity.

## First Session Checklist

- Confirm low-risk audio device access first.
- Test the DAW without third-party plugins before expanding the stack.
- Keep sample libraries and project directories on fast storage.
- Add plugins and external tools gradually and document what remains stable.

## Risks

- Audio workflows become fragile quickly when plugins, drivers, and external interfaces all change at once.
